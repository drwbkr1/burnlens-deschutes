# Portfolio reviewer-experience decision

**Issue / milestone:** #540 / `BL-PORT-001 / P2O4-T36`

**Decision:** `BUILD_ONE_LOCAL_OFFLINE_REVIEWER_PATH`

## Why this is next

The verified Windigo report is truthful and responsive, but it is one isolated
result. A reviewer cannot move from it to the method, retained failures,
reproducibility, case study, or lineage.

The August 6 deadline makes that fragmentation more important than another
source route, governance expansion, or speculative model step.

## Selected outcome

Build one deterministic repository-owned landing page from existing verified
evidence.

The page starts with the strongest result, shows Petes Lake as the visible stop
example, explains source roles and unproven claims, and finishes with exact
lineage and a short quickstart.

The surface is local and offline. It uses tracked assets directly and contains
no network, provider, credential, mailbox, upload, tracking, private-custody, or
deployment capability.

## Analytical boundary

Windigo remains the sixth owner-approved prototype event. The six-event count
minimum is necessary but insufficient. Dataset, split, baseline, and model
remain null.

This decision changes presentation and reviewer navigation only. It does not
change the project promise, target user, CV task, phase outcomes, use
boundaries, source precedence, source roles, owner decisions, labels, or
scientific evidence.
