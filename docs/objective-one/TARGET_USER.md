# BurnLens Deschutes — Target User

## Purpose

This document defines the audience for BurnLens Deschutes without turning the project into a partner, sponsor, or operational pilot.

## Primary portfolio audience

The primary audience is **technical and technical-adjacent portfolio reviewers** evaluating whether BurnLens Deschutes demonstrates credible capability in:

- computer vision workflow design
- geospatial data processing
- map-ready output generation
- reproducibility and versioning
- documentation quality
- responsible AI/GEOINT communication
- practical scoping judgment

These reviewers may include GIS analysts, remote-sensing practitioners, data scientists, resilience technologists, technical hiring reviewers, mentors, or public-interest technology evaluators.

## Reference user persona

The reference user is a **planning-adjacent wildfire resilience reviewer**.

This person is not being treated as a formal partner or external validator. Instead, the persona helps shape what the output should be able to communicate: map context, source notes, uncertainty, limits, and practical screening value.

The reference user needs to understand:

- what area the output covers
- what source data was used
- what method produced the output
- how the output relates to public roads, facilities, parcels, hazard context, or fire-reference layers
- what the output does not prove
- which official sources govern

## User needs

BurnLens Deschutes should help a reviewer inspect whether the workflow is:

- technically coherent
- geographically grounded
- reproducible
- clearly scoped
- transparent about source limitations
- honest about model or baseline uncertainty
- useful as a planning-style screening demonstration

## Audience boundaries

BurnLens Deschutes is not built for:

- live incident commanders
- evacuation decision-makers
- emergency alert operators
- utility operations teams
- insurers or lenders making property determinations
- regulators making official hazard classifications
- the general public seeking current wildfire safety guidance

Those audiences require official, current, operationally governed information beyond this portfolio project.

## Design implication

The project should prioritize clear artifacts over feature volume:

- concise project explanation
- readable maps
- source and run metadata
- visible disclaimers
- traceable outputs
- reproducible commands
- practical summaries
- honest limitations

A reviewer should be able to understand the workflow without private explanation from the builder.
