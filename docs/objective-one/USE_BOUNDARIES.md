# BurnLens Deschutes — Use Boundaries

## Purpose

This document defines what BurnLens Deschutes may and may not be used to represent.

BurnLens Deschutes is a portfolio-first computer vision and GEOINT project. Its outputs are experimental screening artifacts for demonstrating technical workflow design, not official wildfire intelligence or emergency guidance.

## Boundary statement

BurnLens Deschutes may support portfolio demonstration, technical review, and planning-style interpretation of wildfire-relevant imagery and public geospatial layers.

BurnLens Deschutes must not be used or presented as an emergency-response system, evacuation-order tool, incident-command product, official hazard map, field-validated model, or replacement for authoritative public-agency information.

## Appropriate uses

BurnLens Deschutes may demonstrate computer vision applied to wildfire-relevant imagery, geospatial preprocessing, raster-to-vector output conversion, map exports, exposure-style summaries using public layers, reproducible run packaging, provenance tracking, uncertainty communication, and responsible portfolio storytelling around wildfire GEOINT.

These uses are appropriate only when outputs are labeled experimental and supported by source notes, version records, run IDs, and limitations.

## Planning-style observations

BurnLens Deschutes may state what public roads, facilities, or map features are near an experimental output; how a public reference layer overlaps with a study area; whether a run produced a mask, polygon, summary, map export, or warning; and what source dates, processing dates, and quality issues apply.

These observations must be written as screening context, not instructions or determinations.

Preferred language:

- “This output shows...”
- “This run indicates...”
- “This layer overlaps...”
- “This result should be compared with official sources...”

Avoid language such as “safe,” “unsafe,” “evacuate,” “use this route,” “official hazard,” “confirmed fire perimeter,” or “emergency guidance.”

## Prohibited uses

BurnLens Deschutes must not be used for emergency alerts, incident command, suppression decisions, real-time public safety direction, official fire perimeter generation, official hazard classification, parcel enforcement, insurance or lending decisions, utility operational decisions, field-validation claims, agency-endorsement claims, or replacement of official information.

## Source-precedence rule

Official sources govern when BurnLens Deschutes differs from public-agency wildfire, evacuation, emergency-management, hazard, transportation, or incident information.

## Data and model boundary

Model-generated outputs must be labeled:

**Experimental computer vision output. Not field verified. Not official wildfire information.**

Baseline outputs must be labeled:

**Experimental screening output derived from public data. Not official wildfire information.**

## Stop-rule conditions

A BurnLens Deschutes output should be marked **provisional**, revised, or withheld from public display if imagery is stale, source metadata is incomplete, the AOI boundary is unclear, CRS handling is undocumented, cloud/smoke/snow/haze/shadow/sensor issues degrade interpretation, public reference layers conflict, provenance is missing, or the summary cannot be written without overstating what the workflow shows.

## Required disclaimer

**BurnLens Deschutes is an experimental computer vision and GEOINT portfolio project. Outputs are for technical demonstration and planning-style screening only. They are not emergency guidance, official wildfire information, evacuation instructions, or field-validated hazard products. Always rely on official county, state, federal, fire-service, and emergency-management sources for current wildfire and evacuation information.**

## Short UI disclaimer

**Experimental screening output. Not emergency guidance. Not official wildfire information. Check authoritative sources.**

## Portfolio claim boundary

BurnLens may claim reproducibility, traceability, technical workflow demonstration, and transparent limitations when those records exist. It may not claim operational reliability, field validation, agency endorsement, emergency readiness, official risk reduction, live incident accuracy, or decision authority.
