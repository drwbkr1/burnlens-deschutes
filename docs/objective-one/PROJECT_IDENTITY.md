# BurnLens Deschutes — Project Identity

## Public name

**Project display name:** BurnLens Deschutes  
**Short name:** BurnLens  
**Website:** burnlensproject.org  
**Project type:** Computer vision + GEOINT wildfire portfolio project

## Project identity

BurnLens Deschutes is a portfolio-first computer vision and geospatial intelligence project focused on wildfire-related screening in Deschutes County, Oregon.

The project demonstrates how public geospatial data, wildfire-relevant imagery, reproducible processing, and experimental computer vision outputs can be combined into map-ready screening artifacts with clear limitations.

BurnLens Deschutes is not an official wildfire information system. It is a technical portfolio project designed to show practical capability in geospatial processing, computer vision workflow design, versioned outputs, and responsible communication.

## One-sentence description

BurnLens Deschutes turns wildfire-relevant imagery and public geospatial layers into experimental, map-ready screening outputs with traceable methods and strict use boundaries.

## Portfolio thesis

BurnLens Deschutes demonstrates how computer vision and open geospatial data can support transparent wildfire exposure screening by producing traceable maps, summary outputs, and clearly documented limitations for a defined Deschutes County study area.

## Naming rationale

The name **BurnLens Deschutes** combines wildfire focus, image-based analysis, and geographic specificity.

- **Burn** signals wildfire, burn areas, and fire-related landscape change.
- **Lens** signals imagery, computer vision, interpretation, and focused analysis.
- **Deschutes** anchors the project in a bounded geography instead of presenting it as a broad wildfire platform.

The name is intentionally specific. It frames the work as a focused portfolio case study rather than a statewide, operational, or agency-grade wildfire system.

## Public positioning

BurnLens Deschutes sits between abundant wildfire-related data and clear, reproducible planning-style outputs.

The project explores how satellite imagery, public fire layers, local overlays, and computer vision methods can be organized into a transparent workflow that produces maps, summaries, and versioned run artifacts.

The value is not “AI” in the abstract. The value is a disciplined technical workflow: source data, process imagery, generate an experimental layer, connect that layer to geospatial context, record provenance, and explain what the result can and cannot support.

## Technical identity

The intended technical chain is:

**imagery → preprocessing → segmentation or baseline mask → polygons → map overlay → exposure-style summary → documented run package**

The first computer vision direction is a bounded segmentation task, such as active-fire, burn-area, or wildfire-relevant image-mask generation. Outputs should be labeled experimental and traceable to source data, dataset version, model or baseline version, GitHub commit, and run ID.

## Target audience

BurnLens Deschutes is built for portfolio reviewers evaluating technical capability, reliability, transparency, and judgment. The reference user is a planning-adjacent reviewer who needs to understand whether wildfire-relevant imagery and public geospatial overlays can be turned into clear screening products.

## Identity boundaries

BurnLens Deschutes may be described as:

- a portfolio project
- a computer vision and GEOINT workflow
- a wildfire screening demonstration
- a reproducible public-data workflow
- an experimental map and summary generator

BurnLens Deschutes must not be described as:

- an emergency-response system
- an evacuation-order tool
- an incident-command platform
- an official hazard product
- a field-validated wildfire model
- an agency-reviewed or agency-endorsed system
- a replacement for official wildfire or emergency-management information
