# P1O2-T05 — Imagery Assumptions

## Status

Phase 1 / Objective Two task artifact.

This document defines imagery assumptions for the first BurnLens Deschutes CV task. It does not choose final imagery, download imagery, select AOI tiles, preprocess data, create labels, generate a baseline, train a model, or run inference.

## Applies to

Current Objective Two decisions:

- CV task: experimental binary semantic segmentation for wildfire-relevant screening
- Primary target: active-fire / hotspot-informed binary fire mask
- Fallback target: burn-scar binary mask if active-fire label feasibility fails
- Output contract: mask-first output that can later become georeferenced raster, polygons, map overlay, summary, and run package

## Imagery role in BurnLens

BurnLens should treat imagery as the source material for future CV inputs. Imagery is different from reference layers, labels, baselines, planning overlays, and official incident information.

| Category | Role |
|---|---|
| Imagery input | Future model or baseline input tile/scene. |
| Active-fire reference product | Candidate source for baseline, weak labels, sampling, or comparison. |
| Planning overlay | Roads, structures, facilities, boundaries, or other context for later GEOINT integration. |
| Official/public source layer | Authoritative or public reference information that governs interpretation when conflicts exist. |
| BurnLens output | Experimental mask/raster/vector/summary generated by this portfolio workflow. |

## Candidate imagery/source categories

This document keeps several source categories open for Phase Two feasibility review.

| Candidate | Possible role | Why it remains plausible | Key caution |
|---|---|---|---|
| Sentinel-2 MSI | Multispectral imagery candidate for future input tiles. | Sentinel-2 MSI samples 13 bands at 10 m, 20 m, and 60 m resolution and supports land monitoring use cases. | Cloud, smoke, band-resolution differences, revisit timing, and alignment with active-fire references must be handled. |
| Landsat Collection 2 Level-2 | Surface reflectance and/or surface temperature candidate. | Landsat Level-2 includes surface reflectance and surface temperature products relevant to land surface and wildfire-adjacent analysis. | Coarser spatial/temporal fit may limit active-fire segmentation usefulness. |
| NASA FIRMS / MODIS / VIIRS | Active-fire reference, baseline, weak-label, sampling, or comparison input. | FIRMS provides near-real-time active fire/hotspot information from MODIS and VIIRS. | Active-fire detections are not exact segmentation masks; MODIS and VIIRS detections represent pixel centers at different resolutions. |
| Harmonized Landsat Sentinel-2 / Earth Engine catalogs | Possible future access path or harmonized imagery source. | Harmonized sources may reduce friction when comparing Landsat and Sentinel-family imagery. | Catalog-specific QA fields, scaling, masking, and processing history must be documented. |
| Public sample imagery or benchmark wildfire imagery | Early prototype or tutorial input if satellite source feasibility is delayed. | Can help build pipeline logic before full data pipeline complexity. | Must be clearly marked as sample/demo imagery and not presented as Deschutes operational output. |

## Research references

These references justify keeping the above candidates in the Phase Two feasibility queue:

- NASA FIRMS enables access to near-real-time satellite imagery, active fire/hotspots, and related products; global data are generally available within 3 hours of satellite observation, while some U.S. and Canada detections are available in real time. https://www.earthdata.nasa.gov/data/tools/firms
- FIRMS notes that MODIS active fire/thermal hotspot locations represent the center of a 1 km pixel and that VIIRS active fire/thermal hotspot locations represent the center of a 375 m pixel. https://www.earthdata.nasa.gov/data/tools/firms
- NASA Earthdata states that Sentinel-2 MSI samples 13 spectral bands at 10 m, 20 m, and 60 m spatial resolutions. https://www.earthdata.nasa.gov/data/instruments/sentinel-2-msi
- USGS states that Landsat Collection 2 Level-2 includes surface reflectance and surface temperature products, and that surface temperature is useful for monitoring natural disasters including wildfires. https://www.usgs.gov/landsat-missions/landsat-collection-2-level-2-science-products
- Google Earth Engine's Sentinel-2 Harmonized catalog documents QA60 cloud/cirrus mask details and notes changes in QA60 availability and construction. https://developers.google.com/earth-engine/datasets/catalog/COPERNICUS_S2_SR_HARMONIZED

## Required imagery assumptions for Phase Two

Phase Two must evaluate sources against these criteria before any data build:

1. **Public/open access** — data must be accessible through public/open or clearly documented channels.
2. **Licensing clarity** — usage terms must be compatible with a public portfolio repository or clearly documented if not redistributed.
3. **Timestamp availability** — source acquisition time/date must be available or documented as missing.
4. **Spatial reference** — CRS, transform, bounds, and resolution must be available when geospatial outputs are expected.
5. **Band documentation** — band names, wavelengths/use, scaling, nodata, and QA bands must be documented.
6. **Cloud/smoke/quality pathway** — cloud, smoke, obscuration, shadow, snow, water glare, and quality issues must have an exclusion or warning pathway.
7. **Reference alignment** — source imagery must be alignable with reference products well enough for the selected task.
8. **Version/provenance support** — source records must fit the BurnLens versioning and manifest pattern.

## Minimum source metadata required later

Every future imagery record should include at least:

| Field | Meaning |
|---|---|
| `source_name` | Provider/product name. |
| `source_url` | Where the source was accessed or documented. |
| `source_license_or_terms` | License, use terms, or documentation pointer. |
| `acquired_at` | Image acquisition date/time if available. |
| `downloaded_at` | Date/time acquired by BurnLens, if downloaded later. |
| `processed_at` | Date/time processed by BurnLens. |
| `sensor` | Sensor or instrument, if applicable. |
| `platform` | Satellite/platform, if applicable. |
| `band_set` | Bands used or proposed. |
| `spatial_resolution` | Resolution per band where relevant. |
| `crs` | Coordinate reference system. |
| `bounds` | Spatial extent. |
| `qa_fields` | Cloud, quality, confidence, or mask fields. |
| `known_limitations` | Cloud, smoke, stale source, missing bands, alignment risks, or other caveats. |
| `redistribution_status` | Whether the raw/source asset can be committed, linked, or must remain external. |

## Cloud and quality assumptions

Future imagery work must assume that cloud, smoke, snow, shadow, water glare, and other artifacts can make a tile unsuitable for training, validation, inference, or presentation.

BurnLens should not use a tile simply because it is available. A tile must be suitable for the intended purpose and must carry quality notes.

Google Earth Engine's Sentinel-2 Harmonized documentation notes that QA60 cloud-mask handling changed across time and identifies bits 10 and 11 for opaque clouds and cirrus. Phase Two must not treat cloud masking as a generic one-line preprocessing step without documenting the catalog, QA field, and date range used.

## What this document does not decide

This document does not decide:

- final imagery source
- final source access method
- final AOI
- final event/date window
- final band set
- final cloud mask method
- final preprocessing method
- final training split
- final label source

Those decisions belong to Phase Two data feasibility and label planning.

## Use boundary

Imagery-derived BurnLens outputs will be experimental portfolio artifacts only. They must not be used as official wildfire information, emergency guidance, evacuation routing, incident command support, or field-validated hazard assessment.

Official sources govern when information differs.
