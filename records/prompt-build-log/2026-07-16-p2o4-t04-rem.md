## Cycle-start finding

The synchronized v0.12.0 workflow was run before choosing work. Eight of ten outputs reproduced byte for byte. Both GeoTIFF pairs, both HTML pages, and both PNGs were exact. The proposal JSON differed only because OneDrive changed the registered MTBS clips from the previously observed exact-two-link state to the equally approved single-link state; QA JSON then differed only because it binds the proposal hash. Pixel values, grids, source hashes, registered content, diagnostics, decisions, and renders were unchanged.

This made public evidence identity depend on incidental current filesystem topology. It outranked planned label adjudication because the shipped run package was not stable across both topologies the verifier explicitly authorizes.

## Bounded remediation

The runtime verifier remains fail closed. The registration manifest and both MTBS clips must each have one or exactly two links. Exact bytes, sizes, SHA-256, raster driver, CRS, grid, transform, value domain, and counts are still reverified from one bounded read. A third link is rejected.

After that gate passes, public evidence records the policy version, approved counts, policy result, exact manifest hash, and stable asset content checks while omitting the transient observed count. Immutable acquisition-time link trace remains in the preserved registration manifest. No scientific method or public claim changes.

The corrected package uses new `2026-002` report IDs and `r004` run IDs. The v0.12.0 / `2026-001` package remains unchanged as history.
