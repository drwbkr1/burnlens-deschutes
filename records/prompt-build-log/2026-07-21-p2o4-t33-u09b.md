## Transport contract

- Require one exact HTML page and a same-stem sibling JSON output report.
- Require an owner-response schema and matching report/page identity before
  treating a report as an owner-review surface.
- Verify the page and every referenced asset against exact byte-count and
  SHA-256 bindings. A prior report may contribute evidence bindings only when
  the current report binds that report ID and exact report hash.
- Admit the selected HTML as the sole document route. Referenced resources are
  safe sibling static assets only; secondary HTML, JSON/text data, nested
  paths, external/absolute/navigation references, meta refresh, duplicate
  attributes, ambiguous Windows paths, symlinks/reparse paths, and sensitive
  reveal/response/receipt/intake/template/raw/private/adjudication/quarantine
  names fail before bind even when manifest-bound.
- Preload exact bytes once. Later source edits cannot change a running session.
- Bind directly to IPv4 `127.0.0.1`; port `0` delegates one atomic allocation
  to the operating system and an occupied explicit port fails without scan.
- Prefix every route with a random capability token and require the exact
  loopback `Host` header.
- Return only fixed exact-byte responses for `GET` and `HEAD`. Known and unknown
  write/extension methods return sanitized `405` without Python disclosure.
- Provide no directory index, upload, CORS, cookie, source-path log, public
  bind, response store, or server-side response interpretation.
- Send no-store, no-sniff, same-origin, no-referrer, permissions, frame, and CSP
  headers; browser connections are disabled by policy.
- Keep browser opening optional and release the listener under Ctrl+C or a
  browser-open exception.

## Retained findings and remediation

The pre-existing `python -m http.server 8766 --bind 127.0.0.1` served the
repository root broadly. Exact read-only probes returned the repository index,
`.git/config`, project metadata, and raw custody directory listing. The process
was reverified and stopped before the constrained command became the handoff.

Adversarial review reproduced and then closed manifest-bound reveal/private
resource admission, sensitive selected pages, Python-identifying `501` errors
for unknown methods, meta refresh, browser-open listener leakage, duplicate
HTML attribute ambiguity, a missing U09A-to-U09B integration test, and an
atime-sensitive test assertion. All fixes are covered by the final passing
suite. The remaining optional hardening is a dedicated Windows junction test;
the implementation already rejects reparse attributes and the existing
symlink/reparse test passes where the platform permits link creation.
