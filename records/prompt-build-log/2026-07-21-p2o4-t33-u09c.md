## Environment contract

- Pin the managed interpreter to CPython 3.12.10 and declare Python 3.12 or
  newer. The former `>=3.11` metadata was not installable with the existing
  exact NumPy 2.5.1 dependency, which itself requires Python 3.12.
- Keep `runtime` to BurnLens plus its five direct runtime dependencies; add
  only pinned pytest in `dev`; add the pinned vector, projection, raster-array,
  STAC, and cloud-client tools in `geo-research`.
- Check in one `uv.lock` with the complete resolved graph and artifact hashes.
  Setup uses `uv sync --locked`, so stale project metadata fails instead of
  silently installing from an outdated lock.
- Refuse to replace an incomplete or wrong-interpreter `.venv`. Never copy a
  virtual environment between worktrees.
- Hold a path-derived, per-worktree Windows mutex across environment inspection
  and synchronization so two profile changes cannot race over one `.venv`.
- Run `uv pip check` and a matching offline functional verifier after every
  setup. Failure is nonzero and no readiness marker is printed.
- Verify exact direct package versions and real HDF5, image, GeoTIFF, CRS,
  vector overlap/area, GeoPackage, Rioxarray, offline STAC client, and offline
  Boto3 S3-client construction behavior.
- Keep provider URLs, authentication, credentials, data acquisition, source
  approval, label promotion, and data/model authorization outside setup and
  verification.
- Scope the checked-in Codex environment explicitly to Windows PowerShell. It
  runs the same repository-owned setup command and exposes offline verify/test
  actions for newly created worktrees.

## Retained findings and remediation

Initial lock resolution exposed inaccurate Python support metadata rather than
papering over it: the existing exact NumPy version cannot install on Python
3.11. The declaration was narrowed to the truthful lower bound and a concrete
installed interpreter was pinned.

The first dependency audit found disclosed issues in the prior Pillow and
pytest pins. Both were advanced to audit-selected fixed versions, followed by
the full regression, representative-render byte comparison, packaging check,
and a clean final audit.

Parallel verification accidentally invoked two profile synchronizations in the
same isolated worktree. That produced a transient mixed `.venv`, changed no
tracked byte, and was not treated as a package failure. The environment was
recreated from the lock and a per-worktree mutex was added and independently
exercised. The failed concurrent attempt remains evidence for the safeguard.

Independent static review also found that the first setup used `uv sync
--frozen`, which can consume a stale lock. Final setup uses `--locked`; an
actual drifted-project test now proves fail-closed behavior. The review also
prompted structural TOML assertions, explicit Windows scoping, and real
offline STAC/Boto3 client construction instead of import-only checks.
