## Pre-shipment transparency correction

An additional cycle audit found that report schema `0.1.0` tied its `public_metadata_refresh.performed_at_utc` field to the caller-supplied run time even though the deterministic rehearsal performs no provider request. Schema `0.2.0` replaces that field with a fixed `public_metadata_snapshot.observed_at_utc`, a false `live_refresh_performed_by_this_run` flag, and the `ACCESS-2026-005` evidence record. A new regression test proves a 2030 run time cannot inflate the 2026 observation.

A subsequent filesystem audit found that ordinary path checks followed symlinks and allowed multiply-linked files, which could make registered raw bytes alias mutable storage outside quarantine. The validator now rejects link-like quarantine/destination paths and linked asset files before hashing. Contract `v0.2.0` also hashes the complete invariant list together with the three asset records; a regression test proves an invariant change changes the digest.
