## Source role

The official public `USFS_EDW_MTBS_CONUS` ArcGIS ImageServer supplies one bounded annual thematic burn-severity clip for each already-frozen Tepee and McKay event. The source is jointly attributed to the Monitoring Trends in Burn Severity program, U.S. Geological Survey and USDA Forest Service. MTBS is analyst-interpreted remotely sensed reference evidence. It is not field truth, an operational perimeter, an automatic burned-pixel label, or endorsement.

MTBS states that the program maps large U.S. fires, that mapping and analysis are jointly performed by USGS EROS and USDA Forest Service GTAC, and that thematic products use six classes. Its current class descriptions identify code 0 as background/no data, 1 as unburned-to-low, 2 as low, 3 as moderate, 4 as high, and 5 as increased greenness. BurnLens additionally reserves service code 6 as non-processing. The transfer protocol may use codes 2-4 only as supporting evidence; no MTBS value independently creates a candidate label.

## Live service contract

The service snapshot reported version `12`, service item `b84dc1dbb08149aca566adaaa78c1442`, one U8 band, 30 m pixels, minimum/maximum service values 1/6, and nearest-neighbor default resampling. Its selected metadata have canonical SHA-256 `3cf59ccfe434e395aa7705fcaa884354ba238e3db85d1c3015121b020d41ad3c`.

BurnLens requests exact public `exportImage` URLs with an event-year attribute mosaic rule, EPSG:3857 source and image reference, U8 TIFF output, and explicit nearest-neighbor interpolation. Each extent is snapped outward to the service's 30 m grid. The route needs neither credential nor queued email delivery. The full request URL is preserved in the ignored registration manifest and the proposal lineage.

## Storage and verification boundary

The repository is in a OneDrive tree. Both small public TIFFs may be exposed through one transient external OneDrive hard link after BurnLens safely replaces only the repository pathname. The verifier accepts link count 1 or exactly 2 for these two frozen clips only. For the two-link case it performs one bounded read, verifies exact length and SHA-256, and opens the same immutable in-memory byte buffer for driver, CRS, dtype, grid, transform, and complete value-domain checks. No external alias is modified. The four registered Sentinel provider archives retain their strict single-link gate.

## Limitations and drift

The exact clips are annual mosaic extracts, not the fire-level MTBS bundles. The current fire-level portal delivers bundles through an asynchronous email queue and is therefore not a stable byte dependency for this checkpoint. MTBS warns that products can be revised and recommends using the latest available datasets for new work. BurnLens freezes this service/version/byte contract and must create a new source record if later service content or official interpretation changes.
